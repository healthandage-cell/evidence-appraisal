Evidence-Appraisal

An AI skill that appraises clinical protocols and systematic-review manuscripts against the standard reporting and quality tools — plus the worked example it was tested on and a deck explaining it.

Owner: Yvonne  ·  Created: 10 July 2026  ·  Method version: 1.0

This is a self-contained, global capability — not tied to any single project. It bundles PRISMA 2020, PRISMA-NMA, ROBIS, AMSTAR-2, PRISMA-P and SPIRIT into one repeatable, auditable appraisal.

Getting this repo

This repository is the shared, canonical source — grab it here instead of waiting for the tool to be installed in every environment.

  • Download: on GitHub, click Code → Download ZIP, or clone with git clone <repo-url>.
  • Microsoft 365 Copilot → the copilot-m365\ folder (agent instructions + knowledge-checklists.docx).
  • Claude → evidence-appraisal.skill.
  • GitHub Copilot CLI → evidence-appraisal.zip.

Shared openly for the research community under the MIT License — reuse and adapt it freely. Please cite the original checklist publications (PRISMA, ROBIS, AMSTAR-2, etc.) when using them.

The two things you'll do here

Everyday case
1

Appraise a new paper

You don't touch the skill — you use it and record the result.

  1. In Claude, ask it to appraise the document (e.g. "appraise this systematic review against PRISMA and AMSTAR-2").
  2. Save the report into a new subfolder under examples\.
  3. Add a row to examples\appraisal-log.md — date, document, method & tool versions, verdicts. Your audit trail.

No repackaging — you produced an output, not a change to the method.

Occasional
2

Improve the skill itself

When you change how it appraises (add a tool, refine a checklist).

  1. Edit the files in evidence-appraisal\.
  2. Bump version: in SKILL.md and add a CHANGELOG.md entry.
  3. Double-click repackage.cmd (or run python repackage.py).
  4. Re-publish: Claude → Settings → Customize → Skills → Add → select evidence-appraisal.skill.
⚠️ Editing the folder does NOT change the installed skill. The version inside Claude is a snapshot taken at upload time. To publish an update you must re-package (step 3) and re-add it (step 4). Think of the folder as source code and the installed skill as a deployed build.

The key concept the skill encodes

Two different questions, kept deliberately separate:

Is it fully reported?  — PRISMA / PRISMA-P / PRISMA-NMA / SPIRIT (author-side completeness).

Can it be trusted?  — ROBIS & AMSTAR-2 (methodological quality / risk of bias).

A review can be beautifully reported yet still at high risk of bias. A frequent surprise: AMSTAR-2 caps its rating on a single critical item — so a strong review can score "Low" just for omitting a list of excluded studies (see the Di Molfetta example).

Sharing it with a colleague on Copilot

The skill's content is portable — the workflow and checklists are plain text, so they move to almost any AI assistant. How depends on which Copilot she uses; these are different products.

Her environment

Microsoft 365 Copilot (enterprise)

No SKILL.md here — the equivalent is a Copilot agent (instructions + knowledge). A ready-to-use pack is in copilot-m365\:

  1. agent-instructions.md — paste into the agent's Instructions field.
  2. knowledge-checklists.docx — upload as the agent's knowledge (Word, upload-ready).
  3. HOW-TO-build-in-m365.md — step-by-step via the "Create agent" builder or Copilot Studio, with a governance note.
Developer tool

GitHub Copilot (only if she means the coding one)

A different product from Microsoft 365 Copilot.

  1. CLI: supports the Agent Skills format — drop the evidence-appraisal\ folder in its skills directory.
  2. VS Code Chat: convert to .github\prompts\evidence-appraisal.prompt.md, invoked with /evidence-appraisal.

Whichever route: the checklists in references\ are the real asset — make sure they travel with the skill (as the agent's knowledge), not just the workflow.

Quick facts

  • Installed at: Claude → Settings → Customize → Skills → evidence-appraisal (Author: You)
  • Triggers on: requests to appraise / critically appraise / QC a systematic review, meta-analysis, NMA, or study protocol against a checklist or risk-of-bias tool.
  • Routing: SR/MA manuscript → PRISMA 2020 + ROBIS + AMSTAR-2  ·  NMA → adds PRISMA-NMA  ·  SR protocol → PRISMA-P  ·  trial protocol → SPIRIT 2013.
